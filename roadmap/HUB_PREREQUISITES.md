## Side-by-side hub rule

G01 and G02 deploy new foundations beside the currently serving installation.
They must define namespaces/cells, hostnames, databases, secrets, storage,
ownership and GitOps trees that cannot adopt or delete current resources. Traffic
switch happens only after full acceptance, with the previous revision and data
kept as the tested rollback target. Migration/removal of current alpha resources
is a separate explicit transaction after cutover, never an installer side effect.

## Evidence fields

For every row G00 records owner, exact resource class, cost authority, credential
source, data classification, allowed disruption, rollback target, cleanup owner,
expiry and authoritative readback. Values live in protected downstream evidence;
the public ledger stores only sanitized status and proof hashes.
