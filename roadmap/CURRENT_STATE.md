## Public OSS

- `opencloudtech/CloudRING` remote `main` is
  `f4325441e39220d9d99b942524ed3d5d588fa5f1`.
- There are no open public pull requests and 33 open issues.
- The repository contains useful OCS validators and SDK code, OIDC/JWT and IAM
  primitives, a PostgreSQL transactional store, provider-neutral contracts,
  guarded OpenBao operations, backup proof collection and resilience observers.
- It does not yet contain a complete installer, serving identity platform,
  durable provider control plane, executable product registry, production portal,
  billing runtime or real end-to-end reference cloud product.
- The repository's own documentation correctly describes several surfaces as
  contracts, synthetic examples or preflight-only behavior.
- The principal local checkout is divergent from remote `main` and dirty. It is
  not a valid release source. Multiple additional worktrees exist and some are
  dirty; exact counts are volatile and must be generated by G00 for each checkout
  root rather than copied into prose.
- Public `main` has no release tags, so the SBOM/provenance release path has not
  yet been proved on a real release.
- Open issues 81-97 record concrete correctness, installation, storage,
  resilience, IAM, OpenBao, SafePush, OCS and documentation defects. Issues
  23-38 describe the missing provider-control-plane MVP.

## CloudLinux provider and SafePush

- `infra/cloudring_provider` remote `main` is
  `dbc8b5aeb49469ffdf5ee15c0094062c5b72e13b`.
- It contains `cloudring_core`, pinned to public SHA
  `866ec85aaa24accb713f80022cc02d195e2a8ea4`.
- The isolated delivery path currently depends on one project-locked runner
  manager; certificate expiry, SNI, runner liveness and recovery are not yet
  proven as an operated HA service.
- `cloudring_for_everyone` is absent from provider `main`; a removal record and
  guards against reintroduction remain.
- The provider repository explicitly states `PRODUCTION_READY=false` and
  `preflight-and-plan-only`. The target independent bare-metal site currently has
  read-only inventory plus symbolic bindings, not a deployable provider.
- The secondary hosted-metal profile is also contract-only: there is no complete adapter,
  protected live inventory, apply, upgrade/rollback or live acceptance evidence.
- No tracked dependency on Enterprise was found. This invariant must remain
  machine-enforced.

## Immediate implication

The current codebase is not close enough to production to justify adding all
products in parallel. G00 must make delivery truthful and lossless; G01 provides
a disposable development installation; G02 provides the production HA empty
provider; and G03-G11 complete the pre-product kernel and experience. Only then
are reference products added one at a time.
